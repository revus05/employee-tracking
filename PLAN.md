# План разработки Kanban-приложения (Jira-style)

> **Важное примечание для AI/Разработчика:** Использовать только **последние версии** всех зависимостей (Next.js 15+, Bun 1.1+, Tailwind CSS v4, Shadcn UI, Lucide React). 

---

## 🏗 Архитектура и Стек
- **Runtime:** Bun
- **Framework:** Next.js (App Router) + Next API Routes
- **Styling:** Tailwind CSS v4 + Shadcn UI
- **Architecture:** Feature-Sliced Design (FSD)
- **Icons:** Lucide-icons
- **Theming:** Next-themes (Dark/Light mode)

---

## 🛠 Пошаговый план реализации

### 1. Инициализация и настройка окружения
* **Развертывание базы:** Инициализировать проект через `bun create next-app` и настроить структуру папок согласно FSD (app, pages, widgets, features, entities, shared).
* **Конфигурация стилей:** Установить Tailwind v4 и интегрировать Shadcn UI компоненты с поддержкой CSS-переменных.
* **Theme Engine:** Настроить `next-themes` для переключения темной/светлой темы и добавить глобальный компонент `ThemeToggle` с использованием иконок `Sun` и `Moon`.

### 2. Слой Shared и Базовый UI
* **Библиотека компонентов:** Установить и кастомизировать базовые Shadcn компоненты (Button, Input, Dialog, Card, Badge, Popover).
* **Система иконок:** Создать обертку для `lucide-react`, чтобы унифицировать размеры и цвета иконок во всем приложении.
* **Shared API Utility:** Настроить fetch-клиент или экземпляр Axios/TanStack Query для обработки запросов к API Routes с типизацией.

### 3. Авторизация и Ролевая модель (Entity: User)
* **Схема данных:** Создать модели `User`, `Project`, `Task`, `Column` и `Invitation` (Prisma/Drizzle) с четким разделением ролей `MANAGER` и `DEVELOPER`.
* **Auth Backend:** Реализовать регистрацию и логин через Next Auth или JWT в API Routes.
* **Access Control:** Написать Middleware для защиты страниц и API-энпоинтов в зависимости от роли пользователя.

### 4. Управление проектами и Инвайты (Feature: Project Management)
* **Создание проекта:** Разработать форму для менеджера (название, описание) с мгновенным созданием через API.
* **Система приглашений:** Реализовать поиск пользователей по email/username и отправку инвайтов в проект.
* **Принятие участия:** Создать логику для разработчиков: просмотр активных приглашений и вступление в проект.

### 5. Структура Kanban-доски (Entity: Column)
* **Управление колонками:** Реализовать API для создания, редактирования названий и удаления колонок внутри проекта.
* **Кастомизация:** Добавить выбор цвета для каждой колонки (через Popover с палитрой Tailwind) и сохранение состояния в БД.
* **State Management:** Настроить Zustand стор для управления состоянием колонок на фронтенде без лишних ререндеров.

### 6. Задачи и Drag-and-Drop (Feature: Task Management)
* **CRUD Задач:** Создать модальные окна для создания/редактирования задач с полями: исполнитель (Select), дедлайн (Calendar), описание (Textarea).
* **Интеграция DnD:** Подключить `@dnd-kit` для плавного перетаскивания карточек задач между колонками.
* **Визуальные индикаторы:** Использовать иконки `Calendar`, `User`, `AlertCircle` для отображения статуса задачи, исполнителя и дедлайна.

### 7. Аналитический Dashboard (Page: Manager Analytics)
* **Агрегация данных:** Написать API-роуты для подсчета статистики: задачи по стадиям, просроченные (deadline < now), близкие к дедлайну.
* **Визуализация:** Оформить сводные карточки с использованием Shadcn Card и Badge для отображения критических показателей.
* **Лидерборд:** Вывести список самых производительных исполнителей на основе количества выполненных задач.

### 8. Финализация и UX
* **Обработка состояний:** Добавить скелетоны загрузки (Skeletons) для доски и дашборда.
* **Валидация:** Настроить валидацию всех форм на клиенте и сервере с помощью Zod.
* **Dark Mode Check:** Проверить контрастность всех элементов (особенно кастомных цветов колонок) в темной теме.
